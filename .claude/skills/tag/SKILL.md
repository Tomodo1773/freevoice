# Tag スキル

Git タグ（セマンティックバージョニング）を付与し、GitHub にプッシュする。

## バージョン形式

`v{major}.{minor}.{patch}` （例: `v0.3.4`）

- **major**: アプリの機能が大幅に変わるとき
- **minor**: ちょっとした機能追加のとき
- **patch**: バグ修正などのとき

## 手順

1. `git tag --sort=-v:refname | head -1` で最新タグを取得する
2. ユーザーに変更の種類を確認する（major / minor / patch）。ただし、ユーザーが既に指定している場合や、直前の会話から明らかな場合はスキップしてよい。以下を参考に判断する:
   - 「バグ修正」「fix」「修正」→ patch
   - 「機能追加」「新機能」「feat」→ minor
   - 「大幅変更」「破壊的変更」「メジャー」→ major
3. バージョン番号を適切にインクリメントする。インクリメント時、上位が上がったら下位はゼロにリセットする（例: `v0.3.4` → minor なら `v0.4.0`）
4. 新しいタグをユーザーに確認してから付与する:
   ```bash
   git tag v{new_version}
   ```
5. タグを GitHub にプッシュする:
   ```bash
   git push origin v{new_version}
   ```
6. 完了を報告する
