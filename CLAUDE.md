## 言語設定

常に日本語で回答してください。

## 実装方針

- 一般公開するOSSツール。不特定多数のユーザーが利用する前提で品質を担保する
- シンプルさを優先しつつ、堅牢性とのバランスを取る
- YAGNI、KISS、DRYを徹底せよ。
- 状態管理は明示的に行い、暗黙の遷移や競合が起きない設計にする
- エラーハンドリングとセキュリティは適切に実装する
- サイクロマチック複雑度（循環的複雑度）を増大させずに済む場合を除き、後方互換性のためのシムやフォールバックパスは設けないこと。

## テスト方針

- 正常系を中心にテストを実装
- ユーザー操作の競合やタイミング依存の処理は異常系もテストする
- 外部リソース中心でモック中心になるテストは見送る

## アプリ概要

FreeVoice は Windows 専用の音声入力ツール。`Ctrl+Shift+Space` を押している間だけ録音し（Push-to-Talk）、離すと AI で後処理して現在フォーカス中のテキストフィールドに貼り付ける。

## コマンド

```bash
# 開発サーバー起動（フロントエンド + Tauri）
pnpm tauri dev

# プロダクションビルド
pnpm tauri build

# フロントエンドのみビルド
pnpm build

# 型チェック
pnpm tsc --noEmit
```
