# FreeVoice - 要件定義書

> **注意**: このアプリケーションは**個人利用専用**ツールです。開発者自身が日常的に使うために作られており、一般配布・商用利用は想定していません。

Windows向け音声入力アプリケーション。キーボードショートカットで起動し、音声をリアルタイムに文字起こしした後、AIによる誤字脱字修正・フォーマット整形を行い、フォーカス中のテキストフィールドに入力する。

## 1. 機能要件

### 1.1 ショートカットキーによる起動・制御

- システムトレイに常駐し、グローバルホットキーで音声入力を開始する
- **Push-to-Talk方式**: ショートカットキーを**押している間**だけ録音し、**離すと**録音を停止する
- デフォルトのショートカットキー: `Ctrl + Shift + Space`（設定で変更可能）
- どのアプリケーション上でも動作する（グローバルキーフック）

### 1.2 オーバーレイUI（録音インジケーター）

ショートカットキーを押すと、画面下部中央にフローティングオーバーレイが表示される。

#### ステータス遷移

| ステータス | 表示内容 | タイミング |
|-----------|---------|-----------|
| **Listening** | マイクアイコン + リアルタイム文字起こしテキスト | キーを押している間 |
| **Formatting** | スピナーアイコン + 「整形中...」テキスト | キーを離した直後〜テキスト入力完了まで |
| **Done** | チェックマークアイコン（短時間表示後フェードアウト） | テキスト入力完了後 |
| **Error** | エラーアイコン + エラーメッセージ | エラー発生時。5秒後に自動フェードアウト |

#### オーバーレイの仕様

- 常に最前面に表示（Topmost）
- クリック透過（操作の邪魔をしない）
- 半透明の背景に丸角のカプセル型デザイン
- アクティブモニターの下部中央に固定表示
- 短い録音でも長い録音でも邪魔にならないコンパクトなサイズ

### 1.3 リアルタイム音声文字起こし（ストリーミング）

- キーを押している間、マイクからの音声をストリーミングで音声認識APIに送信する
- 認識結果を逐次オーバーレイUIに表示する（ユーザーが認識状況を確認できる）
- 言語は**日本語固定**とする

### 1.4 AI による後処理

キーを離した後、文字起こし結果に対してLLMによる後処理を実行する。

- **誤字脱字の修正**: 音声認識の誤りを文脈から推測して修正
- **句読点の適切な挿入**: 自然な文章になるよう句読点を補完
- **フィラー除去**: 「えーと」「あのー」などの不要なフィラーワードを削除

### 1.5 テキスト入力

- 後処理が完了したテキストを、現在フォーカスしているテキストフィールドに入力する
- 入力方式: クリップボード経由の貼り付け（`Ctrl+V` シミュレーション）

## 2. 非機能要件

### 2.1 信頼性

- 長時間の常駐動作でメモリリークしないこと
- マイクが利用不可の場合、適切なエラーメッセージを表示すること
- APIの通信エラー時にエラーメッセージを表示すること

### 2.2 セキュリティ・プライバシー

- APIキーはローカルの設定ファイルに保管する
- 音声データは処理後にローカルに保持しない
- ローカルにログや音声ファイルを保存しない（デバッグモードを除く）

## 3. 技術構成

### 3.1 アプリケーション基盤

| 項目 | 技術選定 | 理由 |
|------|---------|------|
| フレームワーク | **Tauri** | メモリ効率が良く、Rustバックエンドによりネイティブなシステム統合が可能。Windows専用のため移植性の懸念なし |
| UI | **React** + **TypeScript** | コンポーネントベースで状態管理しやすい |
| グローバルキーフック | tauri-plugin-global-shortcut | 他アプリにフォーカスがあっても動作するキー検知。プラグイン利用のためカスタムRust不要 |
| システムトレイ | Tauri標準API | 常駐アプリとして動作。カスタムRust不要 |

### 3.2 フロントエンド / バックエンドの実装分担

**原則**: ブラウザの標準APIで対応できる処理はすべてフロントエンド（React/TypeScript）で実装する。Rustのカスタムコードは、ブラウザからアクセスできないOSレベルの機能のみに限定する。

| 処理 | 実装場所 | 使用技術 |
|------|---------|---------|
| マイク録音・音声処理 | **フロントエンド** | Web Audio API / MediaRecorder API |
| Realtime APIへのWebSocket通信 | **フロントエンド** | ブラウザ標準 `WebSocket` |
| Chat Completions API呼び出し | **フロントエンド** | `fetch` |
| グローバルホットキー | Tauriプラグイン設定のみ | `tauri-plugin-global-shortcut`（カスタムRust不要） |
| システムトレイ | Tauriプラグイン設定のみ | Tauri標準API（カスタムRust不要） |
| クリップボード書き込み | Tauriプラグイン経由 | `tauri-plugin-clipboard-manager`（カスタムRust不要） |
| Ctrl+Vシミュレーション | **Rustコマンド**（最小限） | `enigo`クレート、`invoke('paste_text')`で呼び出し |
| クリック透過ウィンドウ | **Rustコマンド**（最小限） | Windows API直接呼び出し（ここのみRust必須） |

### 3.3 AIプラットフォーム: Microsoft Foundry

すべてのAIモデルは **Microsoft Foundry（Azure OpenAI）** 経由で利用する。

#### 音声認識: gpt-4o-transcribe

- **モデル**: `gpt-4o-transcribe`
- **接続方式**: WebSocket（Realtime API）によるストリーミング文字起こし
- **エンドポイント形式**: `wss://{endpoint}/openai/v1/realtime?model={deployment}`（APIバージョン指定不要。transcriptionモードはセッション設定で指定）
- **認証**: APIキー（`api-key` ヘッダー）
- **特徴**: GPT-4oベースの高精度文字起こし、従来のWhisperより低いWord Error Rate、多言語対応

#### AI後処理: gpt-5.2

- **モデル**: `gpt-5.2`
- **用途**: 文字起こし結果の誤字修正・フォーマット整形
- **パラメータ**: `reasoning_effort: "none"`（推論なしで高速応答）
- **エンドポイント形式**: `{endpoint}/openai/v1/chat/completions`（APIバージョン指定不要）
- **認証**: APIキー（`api-key` ヘッダー）

### 3.4 テキスト入力シミュレーション

- クリップボード書き込み: `tauri-plugin-clipboard-manager`（フロントエンドからTauriプラグイン経由）
- `Ctrl+V` シミュレーション: Rustコマンド（`enigo`クレート）で実行
- フロントエンドから `invoke('paste_text', { text })` で呼び出す

## 4. ユーザー設定項目

| 設定項目 | デフォルト値 | 説明 |
|----------|------------|------|
| ショートカットキー | `Ctrl+Shift+Space` | Push-to-Talk の起動キー |
| Azure OpenAI Endpoint | - | Microsoft Foundry のエンドポイントURL（例: `https://xxx.services.ai.azure.com`） |
| API Key | - | Microsoft Foundry の APIキー |
| 文字起こしモデル デプロイメント名 | `gpt-4o-transcribe` | 音声認識に使用するモデルのデプロイメント名 |
| 後処理モデル デプロイメント名 | `gpt-5.2` | AI後処理に使用するモデルのデプロイメント名 |

## 6. ユーザー操作フロー

```
1. アプリ常駐中（システムトレイ）
       │
2. ユーザーがテキストフィールドにフォーカス
       │
3. ショートカットキーを押下（KeyDown）
       │  → オーバーレイ表示（Listening状態）
       │  → マイク録音開始
       │  → 音声認識APIへストリーミング送信開始
       │  → 認識結果を逐次オーバーレイに表示
       │
4. ショートカットキーを離す（KeyUp）
       │  → マイク録音停止
       │  → 音声認識ストリーミング終了
       │  → オーバーレイを Formatting 状態に変更
       │  → 最終文字起こし結果をLLMに送信
       │  → AI後処理（誤字修正・フィラー除去）
       │
5. AI後処理完了
       │  → オーバーレイを Done 状態に変更
       │  → 整形済みテキストをフォーカス中のフィールドに入力
       │  → 1秒後にオーバーレイをフェードアウト
```

## 7. 将来的な拡張（スコープ外）

以下はv1のスコープ外とするが、将来の拡張として検討する。

- **言語切替**: 日本語以外の言語対応、自動検出
- **フォーマット整形**: 箇条書き、番号付きリストなど文脈に応じた整形
- **コンテキスト認識**: フォーカス中アプリの画面内容を読み取り、より適切なフォーマットを適用
- **カスタム辞書**: ユーザー固有の専門用語や固有名詞を登録し、認識精度を向上
- **音声コマンド**: 「取り消し」「改行」などの音声コマンドへの対応
- **履歴管理**: 過去の音声入力履歴を保存・再利用
- **ハンズフリーモード**: ダブルタップで起動し、沈黙検知で自動停止
- **翻訳モード**: 日本語で話して英語で入力（またはその逆）
- **音声入力デバイス選択**: 使用するマイクの設定
- **起動時自動開始**: Windows起動時の自動常駐
