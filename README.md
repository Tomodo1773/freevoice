# FreeVoice

Windows 専用の Push-to-Talk 音声入力ツール。ショートカットキーを押している間だけ録音し、離すと AI が自動でテキストを整形してフォーカス中のフィールドに貼り付けます。

---

## 機能

### Push-to-Talk 録音

`Ctrl+Shift+Space` を押している間だけ録音します（キーは設定で変更可能）。録音中は画面下部に半透明のオーバーレイが表示され、現在の状態がひと目でわかります。

### リアルタイム音量メーター

オーバーレイにリアルタイムの音量レベルメーターを表示します。マイクが正常に動作しているか録音中に確認できます。2 秒以上無音が続くと警告を表示します。

### 文字起こしプロバイダーの選択

以下の 2 つの文字起こしプロバイダーから選択できます。

| プロバイダー | 方式 | 特徴 |
|------------|------|------|
| Azure AI Foundry | 録音完了後に一括で文字起こし | `gpt-4o-transcribe` 等のモデルを使用 |
| Azure AI Speech | リアルタイム連続認識 | 録音中に認識結果をオーバーレイにリアルタイム表示 |

### AI 後処理・テキスト整形

文字起こし結果をさらに AI で後処理します。

- 誤字・脱字の修正
- 句読点の自動挿入
- フィラーワード（「えー」「あのー」等）の除去
- 適切な改行の挿入
- プロンプトは設定画面から自由にカスタマイズ可能

### フォーカス中フィールドへの自動貼り付け

整形されたテキストを、現在フォーカス中のテキストフィールドに自動で貼り付けます。貼り付け方式は 2 種類から選べます。

| 方式 | 説明 |
|------|------|
| クリップボード | クリップボード経由で `Ctrl+V` 貼り付け（高速・推奨） |
| キーストローク | 1 文字ずつキー入力をエミュレート（クリップボードに上書きしない） |

### 処理ステータスのオーバーレイ表示

録音から貼り付け完了まで、すべての処理状態をオーバーレイでリアルタイム表示します。

| 状態 | 表示 | 内容 |
|------|------|------|
| 🔴 listening | 録音中 | 音量メーター表示（Azure Speech 時はリアルタイムテキストも表示） |
| ◌ transcribing | 文字起こし中 | AI API 呼び出し中 |
| ◌ formatting | 整形中 | 後処理 AI 呼び出し中 |
| ✓ done | 完了 | テキスト貼り付け完了 |
| ! error | エラー | エラー詳細を表示 |
| 🔇 無音検出 | 音声が検出されませんでした | 1.5 秒間表示後に自動消滅 |

完了・エラー後は 500ms で自動消滅します。処理中（transcribing/formatting）にショートカットを再押下するとキャンセルされ、即座にオーバーレイが非表示になります。

### ショートカットキーのカスタマイズ

デフォルトの `Ctrl+Shift+Space` は、設定画面のキャプチャ機能で好みのショートカットに変更できます。

### スタートアップ自動起動

Windows 起動時にアプリを自動起動する設定が可能です。設定画面の基本設定タブから ON/OFF を切り替えられます。

### ログ出力

設定でログ保存フォルダを指定すると、文字起こし結果・整形後テキストを JSON ファイルとして保存します。ログファイルは日付フォルダ（`YYYY-MM-DD/`）に自動整理され、30 日経過したログは自動削除されます。エラー発生時は常にデフォルト位置にエラーログを出力します。

### システムトレイ常駐

アプリはシステムトレイに常駐します。タスクバーには表示されません。トレイアイコンから設定画面を開いたり、アプリを終了したりできます。

### マルチモニタ対応

オーバーレイはカーソルのあるモニタの中央下部に自動配置されます。

## 動作フロー

### 通常フロー（Azure AI Foundry）

```
Ctrl+Shift+Space 長押し
        ↓
    録音開始（listening）
        ↓
   キーを離す
        ↓
  文字起こし（transcribing）
   Azure AI Foundry API
        ↓
    整形（formatting）
   Azure AI Foundry API
        ↓
 テキスト貼り付け（done）
 フォーカス中フィールドへ
```

### リアルタイム認識フロー（Azure AI Speech）

```
Ctrl+Shift+Space 長押し
        ↓
    録音開始（listening）
  リアルタイム認識が同時に開始
  オーバーレイにテキストを逐次表示
        ↓
   キーを離す
        ↓
    整形（formatting）
   Azure AI Foundry API
        ↓
 テキスト貼り付け（done）
 フォーカス中フィールドへ
```

### キャンセル

処理中（transcribing / formatting）にショートカットを再度押すと、処理を即座にキャンセルしてオーバーレイを非表示にします。

## セットアップ

### 前提条件

- Windows 10 / 11
- マイクデバイス
- Microsoft Azure AI Foundry のアカウントとエンドポイント
- （Azure AI Speech 使用時）Azure AI Speech のエンドポイント

### インストール

[Releases](../../releases) から最新の `.exe` をダウンロードして実行します。

### 初期設定

1. タスクバー右下のトレイアイコンを右クリック → 「設定」を開く
2. **基本設定**タブで**エンドポイント**と **API Key** を入力する
3. 「接続テスト」ボタンで接続確認
4. 「保存」ボタンで設定を保存

設定画面は「**基本設定**」「**プロンプト**」の 2 タブ構成です。

---

## 設定項目

| 設定項目 | デフォルト値 | 説明 |
|---------|------------|------|
| ショートカットキー | `Ctrl+Shift+Space` | 録音開始/停止のキー |
| 文字起こしプロバイダー | Azure AI Foundry | Azure AI Foundry / Azure AI Speech |
| エンドポイント | （空） | Azure AI Foundry のプロジェクト URL |
| API Key | （空） | Azure AI Foundry の API キー（暗号化保管） |
| Speech エンドポイント | （空） | Azure AI Speech 使用時のエンドポイント |
| 言語 | `ja-JP` | Azure AI Speech 使用時の認識言語 |
| 文字起こしモデル | `gpt-4o-transcribe` | 音声認識に使用するモデル |
| 後処理モデル | `gpt-5.2` | テキスト整形に使用するモデル |
| Reasoning Effort | `low` | 後処理モデルの推論強度（none/low/medium/high） |
| 入力方式 | クリップボード | テキスト貼り付け方式 |
| スタートアップ | OFF | Windows 起動時の自動起動 |
| ログ保存フォルダ | （空） | ログ出力先（空の場合は出力なし） |
| フォーマットプロンプト | （組み込み） | AI 整形の指示プロンプト（カスタマイズ可能） |

## 開発

### 必要な環境

- Node.js
- Rust（rustup）
- Tauri CLI v2

### コマンド

```bash
# 開発サーバー起動（フロントエンド + Tauri）
pnpm tauri dev

# プロダクションビルド
pnpm tauri build

# 型チェック
pnpm tsc --noEmit
```

### 技術スタック

| レイヤー | 技術 |
|---------|------|
| フロントエンド | React 18 + TypeScript + Vite |
| UI コンポーネント | Radix UI Themes |
| デスクトップ | Tauri 2 (Rust) |
| 音声認識 | Microsoft Azure AI Foundry / Azure AI Speech |
| テキスト整形 | Microsoft Azure AI Foundry |
