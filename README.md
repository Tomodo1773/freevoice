# FreeVoice

Windows 専用の Push-to-Talk 音声入力ツール。ショートカットキーを押している間だけ録音し、離すと AI が自動でテキストを整形してフォーカス中のフィールドに貼り付けます。

---

## 機能

### Push-to-Talk 録音

`Ctrl+Shift+Space` を押している間だけ録音します（キーは設定で変更可能）。録音中は画面下部に半透明のオーバーレイが表示され、現在の状態がひと目でわかります。

### リアルタイム音量メーター

オーバーレイにリアルタイムの音量レベルメーターを表示します。マイクが正常に動作しているか録音中に確認できます。2 秒以上無音が続くと警告を表示します。

### 文字起こしプロバイダーの選択

以下の 2 つの文字起こしプロバイダーから選択できます。

| プロバイダー | 方式 | 特徴 |
|------------|------|------|
| Azure AI Foundry | 録音完了後に一括で文字起こし | `gpt-4o-transcribe` 等のモデルを使用 |
| Azure AI Speech | リアルタイム連続認識 | 録音中に認識結果をオーバーレイにリアルタイム表示 |

### AI 後処理・テキスト整形

文字起こし結果をさらに AI で後処理します。後処理プロバイダーは **Azure OpenAI** / **OpenAI** から選択でき、文字起こし側とは独立してエンドポイント・API キー・モデルを指定できます。

- 誤字・脱字の修正
- 句読点の自動挿入
- フィラーワード（「えー」「あのー」等）の除去
- 適切な改行の挿入
- プロンプト内の指示文を「整形対象テキスト」と誤認しないよう抑止
- ユーザー辞書・話者ロール（自分の名前や役割）の設定枠を内蔵
- プロンプトは設定画面から自由にカスタマイズ可能
- API 呼び出しに失敗した場合は自動リトライし、最終的に失敗した場合は文字起こし結果をそのまま貼り付けるフォールバック動作

### フォーカス中フィールドへの自動貼り付け

整形されたテキストを、現在フォーカス中のテキストフィールドに自動で貼り付けます。貼り付け方式は 2 種類から選べます。

| 方式 | 説明 |
|------|------|
| クリップボード | クリップボード経由で `Ctrl+V` 貼り付け（高速・推奨） |
| キーストローク | 1 文字ずつキー入力をエミュレート（クリップボードに上書きしない） |

### 処理ステータスのオーバーレイ表示

録音から貼り付け完了まで、すべての処理状態をオーバーレイでリアルタイム表示します。

| 状態 | 表示 | 内容 |
|------|------|------|
| 🔴 listening | 録音中 | 音量メーター表示（Azure Speech 時はリアルタイムテキストも表示） |
| ◌ transcribing | 文字起こし中 | AI API 呼び出し中 |
| ◌ formatting | 整形中 | 後処理 AI 呼び出し中 |
| ✓ done | 完了 | テキスト貼り付け完了 |
| ! error | エラー | エラー詳細を表示 |
| 🔇 無音検出 | 音声が検出されませんでした | 1.5 秒間表示後に自動消滅 |

完了・エラー後は 500ms で自動消滅します。処理中（transcribing/formatting）にショートカットを再押下するとキャンセルされ、即座にオーバーレイが非表示になります。

### ショートカットキーのカスタマイズ

デフォルトの `Ctrl+Shift+Space` は、設定画面のキャプチャ機能で好みのショートカットに変更できます。

### スタートアップ自動起動

Windows 起動時にアプリを自動起動する設定が可能です。設定画面の基本設定タブから ON/OFF を切り替えられます。

### ログ出力・診断ログ

設定でログ保存フォルダを指定すると、文字起こし結果・整形後テキストを JSON ファイルとして保存します。ログファイルは日付フォルダ（`YYYY-MM-DD/`）に自動整理され、30 日経過したログは自動削除されます。

ログ保存フォルダを設定で指定しなかった場合は、デフォルトで以下のフォルダに保存されます。

```
%LOCALAPPDATA%\com.freevoice.app\logs\
```

エラーログや内部診断ログ（異常経路の調査用）も常にこのデフォルト位置に出力されます。

### 履歴ビュー

設定画面の **履歴** タブから、保存済みログを一覧で確認できます。日付・時刻ごとに文字起こし結果と整形後テキストを並べて閲覧できます（ログ保存フォルダ設定時のみ有効）。

### LangSmith トレーシング（任意）

LangSmith の API キー・プロジェクト・リージョンを設定すると、フォーマット呼び出しを LangSmith にトレーシング送信できます。後処理プロンプトの改善やデバッグに利用できます。

### システムトレイ常駐

アプリはシステムトレイに常駐します。タスクバーには表示されません。トレイアイコンから設定画面を開いたり、アプリを終了したりできます。

### マルチモニタ対応

オーバーレイはカーソルのあるモニタの中央下部に自動配置されます。

## 動作フロー

### 通常フロー（Azure AI Foundry）

```
Ctrl+Shift+Space 長押し
        ↓
    録音開始（listening）
        ↓
   キーを離す
        ↓
  文字起こし（transcribing）
   Azure AI Foundry API
        ↓
    整形（formatting）
   Azure AI Foundry API
        ↓
 テキスト貼り付け（done）
 フォーカス中フィールドへ
```

### リアルタイム認識フロー（Azure AI Speech）

```
Ctrl+Shift+Space 長押し
        ↓
    録音開始（listening）
  リアルタイム認識が同時に開始
  オーバーレイにテキストを逐次表示
        ↓
   キーを離す
        ↓
    整形（formatting）
   Azure AI Foundry API
        ↓
 テキスト貼り付け（done）
 フォーカス中フィールドへ
```

### キャンセル

処理中（transcribing / formatting）にショートカットを再度押すと、処理を即座にキャンセルしてオーバーレイを非表示にします。

## セットアップ

### 前提条件

- Windows 10 / 11
- マイクデバイス
- 文字起こし用に **Azure AI Foundry**（`gpt-4o-transcribe` 等）または **Azure AI Speech** のエンドポイント
- 後処理（整形）用に **Azure OpenAI** または **OpenAI** の API キー

### インストール

[Releases](../../releases) から最新の `.exe` をダウンロードして実行します。

### 初期設定

1. タスクバー右下のトレイアイコンを右クリック → 「設定」を開く
2. **モデル設定**タブで文字起こしと後処理それぞれのプロバイダー・エンドポイント・API キー・モデルを入力する
3. 「接続テスト」ボタンで接続確認
4. 「保存」ボタンで設定を保存

設定画面は「**基本設定**」「**モデル設定**」「**プロンプト**」「**履歴**」の 4 タブ構成です。エンドポイントは Azure OpenAI の **v1 形式 URL を直接入力**してください。

## 設定項目

| 設定項目 | デフォルト値 | 説明 |
|---------|------------|------|
| ショートカットキー | `Ctrl+Shift+Space` | 録音開始/停止のキー |
| 入力方式 | クリップボード | テキスト貼り付け方式 |
| 入力デバイス | システム既定 | 録音に使用するマイク |
| スタートアップ | OFF | Windows 起動時の自動起動 |
| ログ保存フォルダ | （空） | ログ出力先（空の場合は出力なし） |
| 文字起こしプロバイダー | Azure AI Foundry | Azure AI Foundry / Azure AI Speech |
| 文字起こしエンドポイント | （空） | Azure AI Foundry の v1 エンドポイント URL |
| 文字起こし API Key | （空） | 文字起こし用 API キー（暗号化保管） |
| Speech エンドポイント | （空） | Azure AI Speech 使用時のエンドポイント |
| 言語 | `ja-JP` | Azure AI Speech 使用時の認識言語 |
| 文字起こしモデル | `gpt-4o-transcribe` | 音声認識に使用するモデル |
| 後処理プロバイダー | Azure OpenAI | Azure OpenAI / OpenAI |
| 後処理エンドポイント | （空） | 後処理用の v1 エンドポイント URL（Azure 選択時のみ） |
| 後処理 API Key | （空） | 後処理用 API キー（暗号化保管） |
| 後処理モデル | `gpt-5.2` | テキスト整形に使用するモデル |
| Reasoning Effort | `low` | 後処理モデルの推論強度（none/low/medium/high） |
| LangSmith API Key | （空） | LangSmith トレーシング用（任意） |
| LangSmith Project | （空） | LangSmith のプロジェクト名（任意） |
| LangSmith Region | US | LangSmith のリージョン（US/EU、任意） |
| フォーマットプロンプト | （組み込み） | AI 整形の指示プロンプト（カスタマイズ可能） |

## 開発

### 必要な環境

- Node.js
- Rust（rustup）
- Tauri CLI v2

### コマンド

```bash
# 開発サーバー起動（フロントエンド + Tauri）
pnpm tauri dev

# プロダクションビルド
pnpm tauri build

# 型チェック
pnpm tsc --noEmit
```

### 技術スタック

| レイヤー | 技術 |
|---------|------|
| フロントエンド | React 18 + TypeScript + Vite |
| UI コンポーネント | Radix UI Themes |
| デスクトップ | Tauri 2 (Rust) |
| 音声認識 | Microsoft Azure AI Foundry / Azure AI Speech |
| テキスト整形 | Microsoft Azure OpenAI / OpenAI |
| トレーシング（任意） | LangSmith |
